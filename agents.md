## 命名规范

### 类名

类名采用**大驼峰命名法（PascalCase）**。

示例：

```cpp
class CaptureManager
{
};
```

---

### 类成员变量

类成员变量采用**下划线开头 + 全小写 + 下划线**的命名方式。

示例：

```cpp
class CaptureManager
{
private:
    int _frame_count;
    bool _is_running;
    std::string _file_name;
};
```

禁止：

```cpp
int frame_count;
int m_frameCount;
int frameCount;
int FrameCount;
```

---

### 成员函数

成员函数采用**全小写 + 下划线**命名方式。

示例：

```cpp
void start_capture();

int get_frame_count();

bool init_encoder();
```

禁止：

```cpp
startCapture();

StartCapture();

GetFrameCount();
```

---

### 局部变量

局部变量采用**全小写 + 下划线**命名方式。

示例：

```cpp
int frame_count;

std::string file_name;

bool is_running;
```

禁止：

```cpp
frameCount;

fileName;
```

---

### 枚举

枚举类型、枚举值统一采用**全大写 + 下划线**。

示例：

```cpp
enum CAPTURE_STATE
{
    CAPTURE_STATE_IDLE,
    CAPTURE_STATE_RUNNING,
    CAPTURE_STATE_STOPPED
};
```

# Android 渲染架构目标

当前 Android 模块正在进行渲染架构升级。

## 当前架构

```
Java
    │
    ▼
Surface
    │
MediaCodec
    │
SurfaceTexture
    │
RenderThread
    │
EGLDisplay
    │
EGLContext
    │
EGLSurface
    │
OpenGL ES
```

## 目标架构

```
Java
    │
    ▼
Surface
    │
JNI AMediaCodec api
    │
TyyPlayer
    │
RenderThread
    │
EGLDisplay
    │
EGLContext
    │
EGLSurface
    │
OpenGL ES
```

最终目标：

- 将java层的opengl和mediacodec解码器都放在Jni层，使用C/C++代替，java传入surface即可。
- 优先支持安卓9系统。
- 使用 C++ 自行管理 EGLDisplay、EGLContext、EGLSurface。
- 使用 C++ 封装一个opengl渲染器类，mediacodec解码器类，然后使用播放器类来进行播放。
- 所有 OpenGL 操作均在 RenderThread 中执行。
- 播放器要求有独立的读取线程，视频解码线程，视频渲染线程。
- 读取线程支持送h264裸流进行解码。
